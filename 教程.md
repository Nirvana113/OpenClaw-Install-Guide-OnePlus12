## 🚀 安装步骤

### 第1步：更新Termux环境

打开Termux，执行：

```bash
pkg update && pkg upgrade -y
```

⚠️ **重要提示**：过程中如提示 Y/N，选 **N** 保留现有配置文件

---

### 第2步：安装依赖

```bash
pkg install build-essential nodejs-lts git cmake python proot-distro -y
```

等待安装完成...

---

### 第3步：安装Ubuntu容器

```bash
proot-distro install ubuntu
```

安装完成后登录Ubuntu：

```bash
proot-distro login ubuntu
```

✅ **成功标志**：提示符变为 `root@localhost:~#`

---

### 第4步：在Ubuntu内更新并安装curl

```bash
apt update && apt upgrade -y
apt install -y curl
```

---

### 第5步：安装Node.js 22

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
```

验证安装：

```bash
node -v && npm -v
```

✅ **成功标志**：应显示 `v22.x.x` 和 `10.x.x`

---

### 第6步：修复Android网络权限问题

Android系统限制proot访问网络接口，需创建补丁文件：

```bash
cat > /usr/local/sbin/fix-network.sh << 'EOF'
#!/bin/bash
# 修复Android网络权限

# 设置网络接口
ip link set lo up 2>/dev/null || true
ip addr add 127.0.0.1/8 dev lo 2>/dev/null || true

# 设置DNS
echo "nameserver 8.8.8.8" > /etc/resolv.conf
echo "nameserver 1.1.1.1" >> /etc/resolv.conf

# 确保网络可用
dhclient eth0 2>/dev/null || dhclient wlan0 2>/dev/null || true

exit 0
EOF

chmod +x /usr/local/sbin/fix-network.sh
```

**设置永久生效**：

```bash
# 添加到启动脚本
echo "/usr/local/sbin/fix-network.sh" >> ~/.bashrc
source ~/.bashrc
```

---

### 第7步：安装OpenClaw

```bash
npm install -g openclaw@latest
```

验证安装：

```bash
openclaw --version
```

✅ **成功标志**：显示版本号

---

### 第8步：创建必要目录和文件

```bash
mkdir -p /root/.openclaw
mkdir -p /root/.openclaw/skills
mkdir -p /root/.openclaw/logs
```

---

### 第9步：运行配置向导

```bash
openclaw onboard --install-daemon
```

**配置选项填写说明**：

| 选项 | 值 | 说明 |
|------|-----|------|
| **Config handling** | Update values | 更新配置值 |
| **Model/auth provider** | Custom Provider | 自定义提供者 |
| **API Base URL** | https://open.bigmodel.cn/api/paas/v4 | 智谱AI API地址 |
| **Endpoint compatibility** | OpenAI-compatible | 直接回车 |
| **Model ID** | glm-4-flash | 临时模型ID |
| **Endpoint ID** | 直接回车 | 略过 |
| **Model alias** | 直接回车 | 略过 |
| **Search provider** | Brave Search | 如有API Key |
| **Configure skills** | No | 不配置 |
| **Telegram** | 填入Bot Token | 从BotFather获取 |

---

### 第10步：配对Telegram用户

启动Gateway：

```bash
openclaw gateway --verbose
```

在Telegram对Bot发任意消息，Bot会返回配对码，例如：

```
openclaw pairing approve telegram NF4RF5T3
```

⚠️ **注意**：在新开的Termux窗口执行，需先 `proot-distro login ubuntu`

---

### 第11步：升级为GLM-4.7-Flash模型

**先验证模型可用**：

```bash
curl -X POST https://open.bigmodel.cn/api/paas/v4/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "glm-4.7-flash",
    "messages": [{"role": "user", "content": "测试"}]
  }'
```

✅ **成功标志**：返回JSON响应，包含模型信息

**更新配置文件**：

编辑 `/root/.openclaw/openclaw.json`：

```json
{
  "agent": {
    "model": "glm-4.7-flash",
    "systemPrompt": "...",
    ...
  }
}
```

## 🛡️ Skills 安装安全规则（永久遵守）

1. **禁止直接修改 openclaw.json** - 安装任何 skills 或插件时，绝对不要手动编辑 openclaw.json 文件
2. **必须备份** - 如果必须配置，先备份 openclaw.json 再操作
3. **使用 CLI 安装** - 正确方式是使用环境变量或 npx clawhub 命令安装
4. **自动加载** - skills 会自动安装到 workspace 的 skills 子目录，OpenClaw 自动加载，无需手动配置
5. **安装命令**：
   ```bash
   # 全局安装 clawhub CLI（首次）
   npm install -g clawhub

   # 安装 skills
   clawhub install <skill-name>

   # 或使用 npx
   npx clawhub install <skill-name>
   ```
6. **禁止直接操作 JSON** - 所有配置必须通过 ClawHub CLI 或环境变量完成
```

**同步修改另外两个文件**：

编辑 `/root/.openclaw/agents/main/agent/USER.md`：

```markdown
name: 主人
称呼: 主人
语言: 简体中文
```

编辑 `/root/.openclaw/agents/main/agent/IDENTITY.md`：

```markdown
name: 团团
nature: AI助手
vibe: 友好、简洁、直接回答问题
emoji: 🤖
owner: 主人
language: 简体中文
```

## 🔧 常见问题

### Q1: 网络连接失败

**A**: 检查科学上网是否正常，确保Termux有网络权限

### Q2: Node.js安装失败

**A**: 更换国内镜像源：
```bash
npm config set registry https://registry.npmmirror.com
```

### Q3: proot网络权限问题

**A**: 重新执行第6步的修复脚本

### Q4: 模型无法调用

**A**: 检查API Key是否正确，确认智谱AI账户有余额

### Q5: Telegram配对失败

**A**: 确保Bot Token正确，重新执行配对命令
