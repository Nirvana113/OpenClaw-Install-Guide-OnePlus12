- **Phone Model**: OnePlus 12 (OnePlus 12)

## 🚀 Installation Steps

### Step 1: Update Termux Environment

Open Termux and execute:

```bash
pkg update && pkg upgrade -y
```

⚠️ **Important**: When prompted Y/N, select **N** to keep existing config files

---

### Step 2: Install Dependencies

```bash
pkg install build-essential nodejs-lts git cmake python proot-distro -y
```

Wait for installation to complete...

---

### Step 3: Install Ubuntu Container

```bash
proot-distro install ubuntu
```

After installation, login to Ubuntu:

```bash
proot-distro login ubuntu
```

✅ **Success Indicator**: Prompt changes to `root@localhost:~#`

---

### Step 4: Update and Install curl in Ubuntu

```bash
apt update && apt upgrade -y
apt install -y curl
```

---

### Step 5: Install Node.js 22

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
```

Verify installation:

```bash
node -v && npm -v
```

✅ **Success Indicator**: Should show `v22.x.x` and `10.x.x`

---

### Step 6: Fix Android Network Permissions

Android restricts proot network access. Create patch file:

```bash
cat > /usr/local/sbin/fix-network.sh << 'EOF'
#!/bin/bash
# Fix Android network permissions

# Set network interface
ip link set lo up 2>/dev/null || true
ip addr add 127.0.0.1/8 dev lo 2>/dev/null || true

# Set DNS
echo "nameserver 8.8.8.8" > /etc/resolv.conf
echo "nameserver 1.1.1.1" >> /etc/resolv.conf

# Ensure network is available
dhclient eth0 2>/dev/null || dhclient wlan0 2>/dev/null || true

exit 0
EOF

chmod +x /usr/local/sbin/fix-network.sh
```

**Set to permanent**:

```bash
# Add to startup script
echo "/usr/local/sbin/fix-network.sh" >> ~/.bashrc
source ~/.bashrc
```

---

### Step 7: Install OpenClaw

```bash
npm install -g openclaw@latest
```

Verify installation:

```bash
openclaw --version
```

✅ **Success Indicator**: Shows version number

---

### Step 8: Create Necessary Directories and Files

```bash
mkdir -p /root/.openclaw
mkdir -p /root/.openclaw/skills
mkdir -p /root/.openclaw/logs
```

---

### Step 9: Run Configuration Wizard

```bash
openclaw onboard --install-daemon
```

**Configuration Options**:

| Option | Value | Description |
|--------|-------|-------------|
| **Config handling** | Update values | Update config values |
| **Model/auth provider** | Custom Provider | Custom provider |
| **API Base URL** | https://open.bigmodel.cn/api/paas/v4 | Zhipu AI API URL |
| **Endpoint compatibility** | OpenAI-compatible | Press Enter |
| **Model ID** | glm-4-flash | Temporary model ID |
| **Endpoint ID** | Press Enter | Skip |
| **Model alias** | Press Enter | Skip |
| **Search provider** | Brave Search | If you have API Key |
| **Configure skills** | No | Don't configure |
| **Telegram** | Enter Bot Token | Get from BotFather |

---

### Step 10: Pair Telegram User

Start Gateway:

```bash
openclaw gateway --verbose
```

Send any message to Bot on Telegram. Bot will return pairing code, for example:

```
openclaw pairing approve telegram NF4RF5T3
```

⚠️ **Note**: Execute in a new Termux window, need to run `proot-distro login ubuntu` first

---

### Step 11: Upgrade to GLM-4.7-Flash Model

**First verify model availability**:

```bash
curl -X POST https://open.bigmodel.cn/api/paas/v4/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "glm-4.7-flash",
    "messages": [{"role": "user", "content": "test"}]
  }'
```

✅ **Success Indicator**: Returns JSON response with model info

**Update config file**:

Edit `/root/.openclaw/openclaw.json`:

```json
{
  "agent": {
    "model": "glm-4.7-flash",
    "systemPrompt": "...",
    ...
  }
}
```

## 🛡️ Skills Installation Safety Rules (Permanent)

1. **Do not directly modify openclaw.json** - When installing any skills or plugins, absolutely do not manually edit openclaw.json file
2. **Must backup** - If configuration is needed, backup openclaw.json first
3. **Use CLI installation** - Correct way is using environment variables or npx clawhub command
4. **Auto load** - Skills will be auto-installed to workspace skills subdirectory, OpenClaw auto loads, no manual config needed
5. **Installation commands**:
   ```bash
   # Global install clawhub CLI (first time)
   npm install -g clawhub

   # Install skills
   clawhub install <skill-name>

   # Or use npx
   npx clawhub install <skill-name>
   ```
6. **Do not directly operate JSON** - All configuration must go through ClawHub CLI or environment variables
```

**Sync and modify other two files**:

Edit `/root/.openclaw/agents/main/agent/USER.md`:

```markdown
name: Owner
nickname: Owner
language: Simplified Chinese
```

Edit `/root/.openclaw/agents/main/agent/IDENTITY.md`:

```markdown
name: Tuantuan
nature: AI Assistant
vibe: Friendly, concise, direct answers
emoji: 🤖
owner: Owner
language: Simplified Chinese
```

### Step 14: Final Startup

```bash
openclaw gateway start
```

✅ **Success Indicator**: Shows Gateway running status

## 🛡️ Prevent Background Process Termination

termux-wake-lock
```

2. Go to **Apps**
3. Find **Termux**
4. Enter **Battery**
5. Select **Unrestricted** (Unrestricted)

## 🔧 Common Issues

### Q1: Network connection failed

**A**: Check if VPN/Proxy is working, ensure Termux has network permissions

### Q2: Node.js installation failed

**A**: Change to domestic mirror:
```bash
npm config set registry https://registry.npmmirror.com
```

### Q3: proot network permissions issue

**A**: Re-run the fix script from Step 6

### Q4: Model cannot be called

**A**: Check if API Key is correct, confirm Zhipu AI account has balance

### Q5: Telegram pairing failed

**A**: Ensure Bot Token is correct, re-run pairing command
